## 📋 File Descriptions

### Backend Files

#### **app/main.py** (FastAPI Application)
- FastAPI initialization
- CORS middleware setup
- Lifespan management (startup/shutdown)
- Health check endpoint
- Root/info endpoint
- **Size**: ~40 lines
- **Purpose**: Application entry point

#### **app/database.py** (Database Configuration)
- PostgreSQL async engine setup
- Connection pooling configuration
- AsyncSessionLocal for dependency injection
- get_db() dependency function
- **Size**: ~30 lines
- **Purpose**: Database connection management

#### **app/models.py** (Data Models)
- Pipeline ORM model (SQLAlchemy)
- PipelineStateHistory ORM model
- AuditLog ORM model
- Pydantic v2 schemas (Request/Response)
- Enum definitions for states and types
- Database indexes and relationships
- **Size**: ~150 lines
- **Purpose**: Data structure definitions

#### **app/routes.py** (API Endpoints)
- POST /api/v1/pipelines - Create pipeline
- GET /api/v1/pipelines - List pipelines (with filtering)
- GET /api/v1/pipelines/{id} - Get specific pipeline
- PATCH /api/v1/pipelines/{id} - Update pipeline state
- DELETE /api/v1/pipelines/{id} - Cancel pipeline
- GET /api/v1/pipelines/{id}/state-history
- GET /api/v1/pipelines/{id}/audit-logs
- GET /api/v1/dashboard/summary
- **Size**: ~250 lines
- **Purpose**: API endpoint definitions

#### **app/auth.py** (Authentication Utilities)
- Password hashing with bcrypt
- JWT token generation
- Token verification
- **Size**: ~40 lines
- **Purpose**: Security utilities (ready for auth implementation)

#### **requirements.txt** (Python Dependencies)
- FastAPI, Uvicorn
- SQLAlchemy, asyncpg
- Pydantic v2
- Password/JWT libraries
- Development utilities

#### **Dockerfile** (Backend Container)
- Python 3.11 slim base
- System dependency installation
- Python dependency installation
- Application setup
- **Purpose**: Container image for deployment

#### **.env** (Environment Configuration)
- DATABASE_URL - PostgreSQL connection string
- SECRET_KEY - JWT secret
- ENVIRONMENT - dev/prod flag
- PORT & HOST - Server configuration
- CORS_ORIGINS - Allowed frontend URLs

## 📝 How to Use This Structure

### For Development
1. Use `start.sh` or `start.bat` to launch
2. Make changes to source files
3. Frontend hot-reloads automatically
4. Backend reloads with `--reload` flag

### For Deployment
1. Push code to GitHub
2. Use `docker-compose` for containerized deploy
3. Or use docker images separately
4. Follow SETUP.md deployment section

### For Maintenance
1. Check logs in Docker or terminal
2. Update dependencies regularly
3. Monitor performance
4. Refer to TESTING.md for procedures
